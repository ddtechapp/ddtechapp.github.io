创业的问题

每天走路上班都在想这个问题，同事说我过于急躁，注意力不够集中，对我自己来说不是什么好事情。我也慢慢认识到这个问题，我太急躁了，想想上次项目确定的时间是八月三十一号，到现在为止已经两个多月了，什么都没有做。我又跑去学习python，跑去参加什么狗屁黑客马拉松，还研究雅思考试，又重头开始学习Java语言，这些事情无可厚非，但我的侧重点在哪里？不想承担责任，但心中想到的东西永远是钱钱钱，因为我缺钱啊！

我创业的初衷就是为了挣钱，无论什么时候这是我的初衷！我们不应该违背自己的初衷，去说一些无关紧要的东西来欺骗自己。成功当然每个人都渴求，成功又不会来的那么容易，我们都不是天才，我甚至都不认为自己是一个人才，连国内的好大学都考不进去，国内的大公司都进不去，芸芸众生罢了。年龄越久，越发现人力的渺小，越来越发现做一件事情的艰难，也越来越对创业有了更多的敬畏。

年轻不懂事，和别人做了很多项目，自己拿了点微博的收入，替别人做嫁衣，还乐呵呵的。现在越来越认清楚生活的真相，不要浪费自己宝贵的时间在一些无所谓的事情上，要学会放手。

大部分人都不是天才，大部分公司也都不是天才创建的，都是十年磨一剑的艰辛。我坚信我会成立一家公司，我也坚信我能够成功。我需要的是沉稳的做事情，慢慢来，不要着急。努力的去解决那些有价值的问题，践行自己的想法，总有一天会成的。

python virtualenv

virtualenv is a tool to create isolated Python environments.

virtualenv是一个python开发工具，按照官方文档来说，这个工具是创建一个不依赖现有电脑环境的独立开发环境。

安装:

sudo pip install virtualenv

创建开发目录:

virtualenv v-env

等待一段时间后，就会看到一个名字为v-env的目录，进入目录里面有三个文件夹: bin, include, lib

bin: 包含python的一些命令工具，python, pip, easy_install等等
include: python的头文件
lib: python的一些lib，这个文件有时间我也会读一读代码的

按照上述文件的内容，我们基本可以知道这里面包含了一个小的python开发环境，按照官方文档的提示，使用

source ./bin/activate

就可以开启开发环境, 使用全局命令deactivate又可以返回现有的环境。

happy hacking, done!

移民

十一后全国开始限购，深圳限购让人感觉有买的起的希望，武汉限购就彻底堵死回去的路了，这样也好，我们不会因为回武汉或者留在深圳而烦恼了。政策已经让我们没有回去的必要了，还是在深圳死磕吧！
最近想做一些事情，规划了一两个月都没有什么起色，让人不禁开始怀疑自己的能力了。或许我只适合做一个不错的工程师，在技术这一条道路上不停地向前发展，而在中国，做技术会意味着道路的变窄，上升通道的过窄会让我这种没什么背景，没钱，没房的人感到深深的恐惧。管理岗位毕竟那么少，能够上去的少之又少。创业概率极低，管理岗概率很低，这两种出路相对中国的房价而言，就不值一提了。剩下还有一个突破就是移民。当然，国外不是天堂，我也不是要追求去享乐，我仅仅是想做一个普通人，过上一种普通人的生活罢了。这种渴求，在中国，是非常昂贵的。像我这种人，在中国几千万，又多少人能够凭借自己的本事，过上一个普通人的生活：一套小公寓，一个老婆，一个孩子，一条狗，一辆车，小孩可以上一个不错的学校。
房地产结合昂贵的公民权去贩卖，房子的价格能够降下来么？没有房子，只能够过二等公民的生活，这是多么的不公平？但，没人去说这不公平，还有很多人嘲笑你没有本事，这就是一个糟糕的现实，你努力工作十四个小时，周末不休息，只能够过一种二等公民的生活，这样真的值得么？这个国家在发展，我们看得到，但这种牺牲老百姓利益的发展，真的公平么？
我不想愤青，也不想骂什么，只是不想接受这种不公平。
想想如何移民吧，对于软件工程师来说，国外会活的更好，做一个技术也会把小孩养的不错。

产品碎碎念

转入做产品已经三个月，还是能够看出有很多不足，不过无论如何我还是不会放弃的，因为我还是想做一些自己的事情。

第一步，成为好的Programmer

第二步，成为好的PM

第三步，成为一名创业者

前进，前进，不择手段的前进…


我们公司的产品有一个很大的问题，这个问题很多人还是看得到的，那就产品线太多，而产品创新能力不足，过于期望开辟

更多的产品线来跑出成绩。资源投入不集中，产品线再多也是白搭，两个人的团队是没有办法和一个二三十人团队竞争的，

连正常的投入都不足于保证，怎么样才能赢呢？

前几个月内心很肿胀，设计了几个产品思路，最后想想也都是自己在家里YY而得的，根本算不上什么太大的创新，无论什么

产品，产品罗辑是很容易自圆其说的，产品的市场在哪里呢？如何去验证？这是非常困难的事情。

今天研究了一下Pocket这个产品，产品定义非常简单，帮助用户保存想要稍后阅读的文章。我只是对他的创始人如何想到这个

产品创意非常感兴趣。按照很多人的想法，这仅仅只是一个功能罢了。但目前来看，pocket的确是一个优秀的产品。在中国做

产品和美国不一样，中国的产品形态是这个产品必须要有一个核心的点，然后围绕这个点区扩展其他功能，后面就形成一个巨无霸

的超级产品，或者叫超级APP，掌握用户流量入口。而美国的很多产品都是在做深度，不做广度，Pocket, Evernote, Dropbox，

Quora，都是在深度上不断地探索，不断挖掘现有创意的价值点，而围绕目前产品的平台型产品，都交给第三方去做。两种做法没有

对错，只是有差异。

而挖掘的价值点是一件非常有难度的事情，如何在现有产品上面去思考更加深入的价值点，还真是产品突破的一个方向。广度是很好

去做的，因为那个更加容易，而深度才是考验人的。

还是要不断学习和思考哦！
